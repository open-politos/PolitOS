# Governance

This directory contains the rules and protocols for democratic decision-making within a PolitOS organization.

## Files

- `voting-rules.yaml` — How votes are conducted, quorum requirements, thresholds
- `deliberation-protocol.yaml` — Phases of discussion before a vote
- `sortition.yaml` — Random selection of decision panels to prevent faction capture
- `constitutional-amendment.yaml` — Higher-threshold process for changing the constitution
- `proposal-lifecycle.yaml` — How a proposal moves from idea to decision

## Design Principles

1. **No rushed decisions** — Every proposal has a mandatory deliberation period
2. **Resistant to capture** — Sortition and randomized panels prevent organized minorities from dominating
3. **Transparent** — Every step is logged in the audit trail
4. **Configurable** — Thresholds, timings, and rules are adjustable per organization
