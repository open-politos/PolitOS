# Getting Started with PolitOS

PolitOS is currently a **specification** — a detailed blueprint for building transparent, AI-augmented political organizations. There is no software to install yet.

## Reading the Specification

The best way to get started is to read through the specification in this order:

1. **[Constitution](../constitution/)** — The hardcoded ethical and legal constraints. Start with `core-principles.yaml` to understand the non-negotiable foundations.

2. **[Governance](../governance/)** — How decisions are made. Read `voting-rules.yaml`, `deliberation-protocol.yaml`, and `sortition.yaml` to understand the democratic process.

3. **[Membership](../membership/)** — Who can participate and what rights they have. See `rights-matrix.yaml`.

4. **[Agents](../agents/)** — The AI components. Start with the `representative/` agent to understand how the AI spokesperson is constrained and governed.

5. **[Config](../config/)** — See `party.config.example.yaml` for how an organization would configure PolitOS for their context.

## Contributing to the Specification

We are actively looking for feedback and contributions. You can:

- Open a [discussion](https://github.com/open-politos/PolitOS/discussions) to ask questions or propose ideas
- Submit a pull request to improve or extend the specification
- Review existing specs and flag gaps or inconsistencies

See [CONTRIBUTING.md](../CONTRIBUTING.md) for guidelines.

## What Comes Next

Once the specification is stable, the next phase will be building implementations — a web platform, AI agent runtime, and voting infrastructure. If you want to help shape that, join the discussion now.
