# Contributing to PolitOS

Thank you for your interest in contributing to PolitOS. This project aims to build open infrastructure for democratic political participation, and it depends on contributions from people with diverse skills and perspectives.

## How to Contribute

### You don't need to be a developer

We need:

- **Political organizers** — Help us design governance rules that work in practice
- **Lawyers** — Help us understand party law and data protection requirements across jurisdictions
- **Designers** — Make governance tools accessible and intuitive
- **Writers** — Translate complex governance concepts for the public
- **Translators** — Help make PolitOS available in more languages

### For developers

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Submit a pull request

### Guidelines

- Keep changes focused and well-documented
- Follow existing file formats (YAML for configuration, Markdown for documentation)
- Every governance rule should have a clear rationale
- Test your changes where possible

## Code of Conduct

- Be respectful and constructive
- Focus on the ideas, not the people
- Remember that PolitOS is ideologically neutral — contributions should not push a specific political agenda into the framework itself

## Questions?

Open a [discussion](https://github.com/open-politos/PolitOS/discussions) or reach out at [open-politos.org](https://open-politos.org).
